Git 常用命令及高级特性

配置

常用命令

1. 初始化仓库

  • **git init**：在项目的根目录运行该命令，会初始化一个新的Git仓库，并创建一个.git目录来存储所有的版本控制信息。

2. 克隆仓库

  • **git clone <repository>**：克隆远程或本地的Git仓库到本地目录。<repository>是仓库的URL。

    Git 拉取代码报错处理：Git Pull Failed: unable to access ‘https://github.xxx.com/hello/Hello.git/‘: SSL certificate problem: self signed certificate in certificate chain.

3. 添加文件到暂存区

  • **git add <file>**：将指定的文件添加到暂存区（也称为索引或暂存空间），准备进行下一次提交。

4. 提交更改

  • **git commit -m "<message>"**：提交暂存区中的更改到本地仓库，并附带一条提交信息<message>来描述此次更改。

5. 查看仓库状态

  • **git status**：显示当前仓库的工作目录和暂存区的状态，包括哪些文件被修改了、哪些文件被添加到暂存区了等。

6. 查看提交日志

  • **git log**：显示提交历史记录，包括每次提交的哈希值、作者、日期和提交信息。

7. 推送更改到远程仓库

  • **git push <remote> <branch>**：将本地分支的更改推送到指定的远程仓库和分支上。<remote>是远程仓库的名称（如origin），<branch>是分支名称。

8. 从远程仓库拉取代码

  • **git pull <remote> <branch>**：从远程仓库拉取最新的代码和更改，并合并到本地分支。

9. 分支管理

  • **git branch**：列出所有本地分支。
  • **git checkout <branch>**：切换到指定的分支。
  • **git merge <branch>**：将指定的分支合并到当前分支。

高级特性

1. 分支与合并

  • **git rebase**：将一个分支的更改“重放”到另一个分支的顶部，从而创建一个更加线性的提交历史。
  • **git cherry-pick**：将特定的提交从一个分支应用到另一个分支上，而不需要合并整个分支。

2. 提交操作

  • 交互式rebase（git rebase -i）：允许用户以交互方式重新排列、合并或编辑多个提交。
  • **git commit --amend**：修改最后一次提交的信息或内容。

3. 子模块（Submodules）

  • **git submodule**：允许将一个Git仓库作为另一个Git仓库的子目录。

4. 钩子（Hooks）

  • Git钩子：在特定事件（如提交、推送等）发生时自动执行的脚本，允许插入自定义的脚本以自动化常规任务。

5. 高级查看与搜索

  • git log 的高级用法：使用各种选项和参数来过滤和格式化提交日志。
  • **git bisect**：二分查找引入错误的提交。

6. 暂存与恢复

  • **git stash**：将当前工作区的更改暂存起来，以便稍后恢复。
  • **git stash pop**：恢复之前暂存的更改并删除暂存记录。

7. 远程仓库操作

  • **git remote**：管理远程仓库的URL和其他信息。
  • **git remote set-url [remote-name] [new-url]**：修改远程仓库的名称和URL。
  • **git remote -v**：列出所有远程仓库的名称和它们对应的URL。
  • **git fetch**：从远程仓库获取最新的更改，但不自动合并到当前分支。
  • **git pull --rebase**：拉取远程仓库的更改并使用rebase来合并，以保持线性历史。

8. 高级配置

  • **git config**：配置Git的各种选项，如用户名、邮箱、别名等。

9. 高级冲突解决

  • Git提供了强大的冲突解决机制，允许在合并或rebase过程中解决冲突。通过git mergetool等命令，可以使用图形界面或手动编辑文件来解决冲突。

10. 多仓库提交

  • **git remote add [remote-name] [remote-url]**：单个项目同时设置多个仓库地址。
