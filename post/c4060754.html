安装完成后，运行预先安装了 MySQL 的安全脚本。

系统会询问您是否要配置VALIDATE PASSWORD PLUGIN

注意： 如果启用，与指定条件不匹配的密码将被MySQL拒绝并出现错误。 虽然禁用验证是安全的，但建议使用强大唯一的密码。

按 Y 设置或者其他按键继续而不设置。

如果回复 Y，则会要求选择一个密码验证级别。

无论您是否选择设置 VALIDATE PASSWORD PLUGIN ，服务器都会要求选择并确认MySQL root 用户的密码。 这是MySQL中具有增加权限的管理帐户。

如果启用了密码验证，则会显示 root 用户的密码强度，是否还需要更改？

对于其余的问题，请在每个提示下按Y并ENTER按键。这将删除一些匿名用户和测试数据库，禁用远程root登录，并加载这些新规则，以便MySQL立即尊重您所做的更改。

注意，在运行MySQL 5.7（及更高版本）的Ubuntu系统中，默认情况下，MySQL 根用户设置为使用auth_socket插件而非密码进行身份验证。

如果希望 root 使用密码连接，则需要将其身份验证方法从切换auth_socket为mysql_native_password。
