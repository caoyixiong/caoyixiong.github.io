Go 常用命令

发表于2024-08-20|更新于2024-09-03|Go
|字数总计:1.4k|阅读时长:5分钟|阅读量:

Go语言（通常称为Golang）是一种由Google开发的静态类型、编译型、并发型，并具有垃圾回收功能的编程语言。在Go语言的开发过程中，掌握一些常用的命令行工具及其命令是非常重要的。以下是一些Go语言开发中常用的命令及其详细解释。

查看全部命令

查看用法

1. 环境安装与配置

go install

  • 用途：安装编译后的包和依赖到$GOPATH/bin目录下，或者指定的GOBIN目录下。
  • 示例：go install example.com/user/hello 会编译hello包及其依赖，并将结果可执行文件安装到$GOPATH/bin或$GOBIN。

go env

  • 用途：打印Go语言的环境信息，如GOPATH、GOROOT、GO111MODULE等。
  • 示例：go env 列出所有环境变量。

2. 代码构建与测试

go build

  • 用途：编译Go程序，生成可执行文件（在Windows下是.exe文件）。
  • 选项：
    • -o：指定输出文件名。
    • -v：显示详细编译信息。
  • 示例：go build -o myapp main.go 编译main.go并生成名为myapp的可执行文件。

go test

  • 用途：运行Go语言编写的测试代码。
  • 选项：
    • -v：显示详细测试信息。
    • -run：指定运行哪些测试函数。
  • 示例：go test -v ./... 在当前目录及其子目录下运行所有测试，并显示详细输出。

go run

  • 用途：编译并运行Go程序，但不生成可执行文件。
  • 示例：go run main.go 编译并运行main.go。

3. 包管理与依赖

go get

  • 用途：下载并安装依赖的Go包。
  • 选项：
    • -u：更新已存在的包到最新版本。
    • -d：只下载包，不安装。
  • 示例：go get -u github.com/golang/dep/cmd/dep 下载并更新dep工具。

go mod

  • 用途：Go 1.11版本引入的模块依赖管理工具。
  • 常用命令：
    • go mod init [module]：初始化新的模块，创建go.mod文件。
    • go mod tidy：添加缺失的依赖，并移除未使用的依赖。
    • go mod vendor：将依赖复制到vendor目录。
    • go mod download：下载并安装模块依赖，但不修改go.mod。

4. 文档生成

go doc

  • 用途：显示Go包的文档。
  • 示例：go doc fmt Println 显示fmt包中Println函数的文档。

godoc

  • 用途：启动一个本地的Web服务器，通过Web界面查看Go的文档。
  • 示例：在命令行中运行godoc -http=:6060，然后在浏览器中访问http://localhost:6060查看文档。

5. 其他常用命令

go fmt

  • 用途：格式化Go代码，使其符合Go的编码规范。
  • 示例：go fmt ./... 格式化当前目录及其子目录下的所有Go文件。

go list

  • 用途：列出安装的包信息。
  • 选项：
    • -m：列出模块路径。
    • -f：自定义输出格式。
  • 示例：go list -m -f '{{.Dir}}' all 列出所有依赖的模块路径。

go clean

  • 用途：删除编译生成的文件。
  • 选项：
    • -i：删除安装的包和依赖。
    • -n：打印将要删除的文件，但不实际删除。
  • 示例：go clean -i 删除安装的包和依赖。

掌握这些Go语言的常用命令，将大大提高你的开发效率和代码质量。随着Go语言的不断发展和更新，建议定期查阅最新的官方文档以获取最新的信息和最佳实践。
